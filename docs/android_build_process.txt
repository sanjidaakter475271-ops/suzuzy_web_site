# Android Build Export Process using GitHub Actions
# Project: Royal Suzuky (ServiceStuff Mobile App)
# Date: 2026-03-08

This document outlines the step-by-step process of exporting the ServiceStuff (Capacitor/React) app to an Android APK using GitHub Actions without installing Android Studio locally.

---

### 1. Prerequisites (What you need)
- Project source code on GitHub.
- GitHub Secrets configured (see step 2).
- Java 21 compatibility (configured in workflow).

---

### 2. GitHub Secrets Setup
Before building, you MUST add the following secrets to your GitHub repository (Settings > Secrets and variables > Actions > New repository secret):

| Secret Name | Example/Purpose |
|-------------|-----------------|
| `VITE_PORTAL_API_URL` | https://royal-suzuky-portal.onrender.com |
| `VITE_REALTIME_URL` | https://royal-suzuky-realtime.onrender.com |
| `VITE_SUPABASE_URL` | Your Supabase Project URL |
| `VITE_SUPABASE_ANON_KEY` | Your Supabase Public/Anon Key |
| `VITE_GEMINI_API_KEY` | Your Google Gemini API Key |

### 3. Workflow File Configuration
The build is controlled by the file: `.github/workflows/android_build.yml`.
Key configuration:
- **JDK Version:** 21 (Required for modern Capacitor/Android builds).
- **Trigger:** Automatic on `push` to `main` branch if files in `servicestuff/` are changed.
- **Manual Trigger:** Can be run manually from GitHub Actions tab (via `workflow_dispatch`).

---

### 4. How to Build and Export
Whenever you need a new APK:

1. **Commit your changes:**
   ```bash
   git add .
   git commit -m "Your update message"
   ```

2. **Push to GitHub:**
   ```bash
   git push origin main
   ```

3. **Monitor the Build:**
   - Go to your GitHub repository in the browser.
   - Click the **"Actions"** tab.
   - Click on the latest running workflow (e.g., "Build Android APK").
   - Wait ~5 minutes for it to complete.

4. **Download the APK:**
   - Once the build finishes with a green checkmark (✅), scroll down to the **"Artifacts"** section.
   - Click on **"app-debug"** to download the zip file.
   - Extract the zip to find your `app-debug.apk` file.

---

### 5. Troubleshooting
- **Error: "invalid source release: 21"**: This means the Java version in the workflow (setup-java) does not match the project requirements. Ensure it is set to `java-version: '21'`.
- **Error: "failed to push some refs"**: This happens when your local code is behind GitHub. Use `git pull origin main --rebase` before pushing.
- **App cannot connect to Server**: Ensure your GitHub Secrets have the correct production URLs and that you are using HTTPS for Render URLs.
